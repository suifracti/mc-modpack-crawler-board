# MC 整合包爬虫看板

把 [MC百科](https://www.mcmod.cn/) 上的整合包信息抓到本地，再生成一个 **双击就能开的 HTML 看板**，方便自己搜、筛、比、看趋势。

目前只接了 MC百科。名字里的「多平台」是以后想往外扩的方向，不是已经全接好了。

---

## 你能拿它干什么

- **找包**：按名字、介绍、评论、标签、包含模组搜索  
- **筛包**：分类、标签、模组类型、具体模组、趋势天数  
- **排序**：指数、浏览、趋势、红黑票等互动数据  
- **看走势**：表格里有迷你趋势图，悬停看更长时间线（本地会攒比官网更长的历史）  
- **扫介绍 / 评论**：悬停名称看介绍，悬停互动区看已抓评论，可跳回百科原页  
- **按模组反查**：展开模组列表，点某个模组就只看带它的整合包  

数据都在你电脑上，看板也是本地 HTML，不依赖在线服务。

---

## 怎么用

环境：本机装好 **Python 3**，爬虫还需要能跑浏览器自动化（项目里用的那套登录/抓取环境）。

在项目目录打开终端：

```powershell
# 1）抓数据（首次会比较久；之后可增量）
python "多平台聚合爬虫_v1.0.py"

# 只刷新超过 N 天没更新过的
python "多平台聚合爬虫_v1.0.py" --refresh-days 1

# 强制全刷（慢，慎用）
python "多平台聚合爬虫_v1.0.py" --refresh-all

# 2）生成看板
python "多平台聚合转换器_v1.0.py"

# 换主题（可选：warm / dark / light / eye / pink / anime）
python "多平台聚合转换器_v1.0.py" -t dark
```

然后双击打开 **`多平台聚合看板_V1.0.html`** 即可。

日常更新就两步：**爬一次 → 转一次 → 刷新浏览器**。

想试生成结果又不想盖掉正式看板时：

```powershell
python "多平台聚合转换器_v1.0.py" -o "ignored_local_files\tmp_check.html"
```

## 目录里主要是啥

| 文件 / 目录 | 说明 |
| --- | --- |
| `多平台聚合爬虫_v1.0.py` | 从 MC百科抓整合包、评论、趋势、包含模组 |
| `多平台聚合转换器_v1.0.py` | 把数据打成 HTML 看板 |
| `多平台爬虫数据_v1.0.jsonl` | 当前数据快照 |
| `trend_history/` | 本地长期趋势（越跑越长） |
| `多平台聚合看板_V1.0.html` | 日常打开的看板入口 |
| `ignored_local_files/` | 本机临时文件、浏览器登录状态等 |

更细的文件归类见 `LOCAL_FILE_MAP.md`（可选看）。

## 注意

- `ignored_local_files/browser_data/` 里是爬虫用的浏览器登录状态和 Cookie，属于账号隐私，别发给别人或随处备份上传。
- 主页面可直接双击离线打开；评论和完整模组详情分别放在 `data/comments/`、`data/mods/`，只在打开对应区域时才读取，减轻 Windows 首次打开压力。评论若还在补抓，显示不完整是正常情况。
- 按需读取不会关掉悬浮预览；同一条内容读过一次后会留在当前页面内存里，再悬停会直接显示。
- 反馈表单草稿会自动保存在浏览器本地，刷新或误关页面后还能恢复；提交成功后自动清除。
- 右下角有「意见反馈」。部署说明和 Cloudflare Worker / Google Apps Script 模板在 `feedback/`；部署好后把 Worker 地址填进转换器顶部的 `FEEDBACK_URL`。
- 评论逐条跳转依赖 MC百科页面是否提供稳定锚点；站点没给单条评论链接时，看板只会保留原页面入口。
- 数据仅用于个人整理与学习交流，排序和评分不代表作品质量结论。
